Additional correctness fixes from this pass:
- worldgen local water IDs now match Java 1.2.5: moving water=8 and still water=9;
- water lakes now place still water ID 9;
- water springs/desert wells now place moving water ID 8;
- cave liquid pre-checks now detect both moving/still water and moving/still lava;
- pumpkin worldgen now writes facing metadata instead of dropping the random value;
- decorator order now follows ChunkProviderGenerate.populate -> BiomeDecorator.decorate more closely.

Limitations:
- This confirms numeric ID and texture/icon mapping coverage. It does not mean every block or item has full Java gameplay behavior.
- Full visual parity still depends on using a compatible Java 1.2.5 terrain.png/items.png atlas through the existing asset pipeline.
